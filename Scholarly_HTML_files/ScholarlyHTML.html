1. Introduction

1.1 Purpose

The purpose of this document is to present in detail the web application Boo. This page will explain the scope and features of the software, as well as its interfaces, functionalities and constraints. Users of this website are free to use this document as a source of information.

1.2 Document Conventions

This document was created based on the Scholarly HTML document format and IEEE template for System Requirement Specification Documents.

1.3 Intended Audience and Reading Suggestions

The users intended to use this web application are those passionated about reading books and reviewing them. If they wish, they can use this document as a tutorial for the site but this is only optional.

1.4 Product Scope

Boo has been developed as the perfect online space for readers all around the world. Its purpose is to connect people who are interested in reading, allow them to share their opinions on any book, organize their reading list and so on.

1.5 References

In order to create the web application and this document we did a research on CSS and HTML related sites which are shown below:

  • W3schools
  • IEEE-Tempate
  • Scholarly HTML
  • Mozilla

2. Overall Description

2.1 Product Perspective

Boo allows its users to share their progress and reviews on the books they are reading, no matter if the user is a single person or a group (a club, a classroom).

2.2 Product Functions

The application has different functionalities and properties like:

  • the posibility to create a profile in order to save each user's preferences
  • a navbar that permits the users to easily access the pages of the site
  • the possibility to add books to customized lists
  • the possibility to search for any book
  • checking friends activity via feed
  • the possibility to access all the books and their information stocked in the database

2.3 User Classes and Characteristics

The users using the the website are the ones looking to connect to other people who love reading, or the ones who just want to become more organized regarding their reading list.

2.4 Operating Environment

The web application can be used on any working device with a web browser and internet connection.

2.5 Design and Implementation Constraints

Boo will be designed using HTML, CSS, JavaScript and will have a server made in Java and also a database created in SQLite.

3. External Interface Requirements

3.1 User Interfaces

3.2 Hardware Interfaces

The devices which will support the website just need an working browser and internet connection.

3.3 Software Interfaces

The application will have a SQLite database and a Java server which will communicate with the interfaces using APIs.

3.4 Communications Interfaces

The communication standard used is HTTP. The book list will be saved in documents formatted PDF and JSON and these will be send through APIs to the server or the database and vice versa.

4. System Features

4.1 Home page

Homepage gives a few fun facts about the site. Here, the user can choose to Sign in, Sign up, read more about the site or access the Help page.

4.2 Sign in page

Users which already have an account on the app can login by entering their credentials and access the informations of the site. Otherwise, users can read about the site and click on the 'create your account' button.

4.3 Sign up page

Users who wish to create an account in order to have access to the site can enter the desired credentials and if they are not taken, a new account will be created for the user's use.

4.4 About page

This page will present general information about the website and what the users can expect to do using it.

4.5 Help page

This page allows the user to ask any questions regarding Boo.

4.6 Forgot password page

This page allows an already existing user to change his password.

4.7 Terms and conditions page

This page contains the terms and conditions.

5. Other Nonfunctional Requirements

5.1 Safety Requirements

The web application may only suffer from a database loose of information, which is highly unlikely.

5.2 Security Requirements

The application will cotain SQL injection protection so that the informations can't be stolen.
